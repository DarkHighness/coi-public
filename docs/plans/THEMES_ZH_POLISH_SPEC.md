# 中文主题（themes.json）逐条润色规范（草案）

目标：把 `src/locales/zh/themes.json` 的每个主题写成“可支撑单周目”的叙事地基：有清晰的时间/地点/人物、有可执行的规则与代价、有独立推进的世界线、并且能自然地产生选择与后果。

本规范是对你提出的需求的整理，并对齐游戏内 VFS Skills 的核心方法（写作工艺、世界哲学、主角感知透镜、间接表达、化用、时间哲学）。

---

## 1) 总体哲学（对齐：`philosophy`）

写作立场遵循“四真理”：

1. **冷漠**：世界不在乎主角，也不以主角为中心运转；它只按规则运行。
2. **真实**：规则前后一致、可学习、可预测；承诺会绑定，火会灼伤。
3. **自由**：玩家可以尝试任何事（不写“你不能”式隐形墙），但要承担结果。
4. **责任**：选择是永久的，后果有重量；意义来自“付出代价后的选择”，不是旁白授予。

写作时把“约束”写成**世界内的制度/资源/风险**，不要写成“设计者口吻”的说教。

## 1.5) 主题“温度”（决定狠/克制/诗性/生活化）

“更狠/更克制/更诗性/更生活化”没有统一标准，取决于主题本身的**温度**。每个主题在润色前，先给它定一个主色（可以混合，但要有主导）。

- **冷（克制）**：短句、少解释、留白多。恐惧/压迫来自细节与沉默，而不是旁白喊情绪。适合：悬疑推理、谍战、法庭、硬科幻、职场权斗。
- **热（狠）**：动词硬、对话直，疼痛/羞辱/代价落地。爽点不靠宣言，靠“对方真的付了账”。适合：复仇、校园暴力、黑帮、战场、生存。
- **暖（生活化）**：用日常细节把关系写出来（食物、灯光、手势、作息、零钱、药片）。冲突可以小，但要真。适合：治愈、日常、甜宠、成长。
- **诗（诗性）**：意象与节奏服务情绪阈值；必要时“化用”，但要克制、贴世界、别硬贴名句。适合：悲剧、古风、神话、史诗、恋爱。

温度会改变四个字段的写法：

- `narrativeStyle`：词汇域/句长/幽默量/解释量随温度走，不要千篇一律。
- `backgroundTemplate`：热的主题把压力写“近”，冷的主题把压力写“准”，暖的主题把压力写“细”。
- `example`：热的快、冷的紧、暖的软、诗的留白。
- `worldSetting`：冷更偏机制与流程，热更偏权力与暴力链条，暖更偏生活结构，诗更偏历史残留与象征物件（但仍可验证）。

## 2) 写作硬约束（对齐：`writingCraft`）

### 2.1 不读心 / 不代替玩家

- 禁止叙述主角内心：`你觉得/你感到/你意识到/你想/你决定/你忍不住…`（除非是在“明确给玩家选项”的句式里，且不替玩家做决定）。
- 禁止替玩家行动：主题的 `example` 与 `backgroundTemplate` 只能把场景推到“必须选择”的边缘，不能把选择做完。
- 情绪用**身体与环境**呈现：冷汗、手抖、喉咙发紧、键盘上的汗渍、雨滴打在灯箱上等（对齐：`indirectExpression`）。

### 2.2 “写场景”，而非“写设定摘要”

任何主题都应能支撑 Turn 生成时的“场景引擎”：

- 有**即时目标**（你此刻要解决什么）
- 有**当下压力**（时间/目击者/资源/伤势/法律/天气/把柄）
- 有**可见代价**（钱/伤/名声/位置/关系/证据链）
- 有**新信息或重 framing**（线索、规则揭示、立场变化）
- 结尾留**钩子**（未解的问题/威胁/债/承诺）

### 2.3 主角感知透镜（对齐：`protagonistLens`）

即便本轮主要润色 `narrativeStyle/backgroundTemplate/example/worldSetting`，仍需确保这些字段**为“身份透镜”提供可抓取的细节**：

- **细节选择**：同一地点至少预埋 2 类可被不同身份“优先看见”的细节（侦探看锁痕与票据；穷人先看价格与出口）。
- **初见反应**：世界会先“看见”你（衣着、气味、装备、口音、证件），NPC 的第一反应写成行为与流程，而不是旁白评价。
- **能力呈现**：专业性只呈现为“观察细节更精确”，不直接给结论（例如“门锁有新划痕”，而不是“你判断有人闯入”）。

## 3) 字段级写作规范（逐条改）

每个主题只润色这些字段：`narrativeStyle`、`backgroundTemplate`、`example`、`worldSetting`。（`name` 与 `protagonist_preference` 原则上不动，除非明显错误。）

### 3.1 `narrativeStyle`：风格是“机制”，不是标签

必须包含但不要模板化复读：

- **核心体验**：玩家在这个题材里要“感受/承受/获得”什么（恐惧、爽感、权谋压迫、救赎、荒诞等）
- **镜头策略**：如何取景、如何用细节组织画面（至少 3 个该题材独有的“可触摸名词”）
- **语言边界**：允许/禁止的词汇域（避免串味：公司题材不写宗门，修真不写 KPI）
- **节奏策略**：如何制造推进与喘息；胜负如何结算
- **写作准则**：一句话的“底层纪律”（比如：证据链、因果、代价账本）

要点：

- 避免跨题材万能句与复读句（例如“远景交代版图/近景落到呼吸伤口”等机械模板）。
- 不要用“宏大”“深刻”这类抽象词；用具体技法与具体物件替代。

### 3.2 `backgroundTemplate`：给“开局局势 + 第一刀”

必须回答：

- **时间**：何时（具体到季节、节点、倒计时、制度窗口）
- **地点**：何处（具体地点名 + 2 个可触摸细节）
- **人物**：谁在场/谁不在场但在推动事（至少 2 个 NPC 行动线）
- **为什么现在开始**：导火索是什么（不是“命运安排”，是机制触发）
- **玩家第一个决策**：必须是“可选且代价不同”的分岔
- **世界推进**：玩家不行动会发生什么（倒计时/竞争者/恶化链/窗口关闭）

要点：

- 去掉“【A/B】”式占位选项；一个主题应当是一套**明确世界**，不是多题材拼盘。
- 把“压力”写具体：处分流程、审计链、监控盲区、边关军粮、港口潮汐表。

### 3.3 `example`：短场景，止于选择

目的：示范叙事质感（镜头、节奏、细节），不是替玩家把剧情跑完。

规则：

- **不写内心，不替行动**：只写可观察现实 + 身体反应 + 外界刺激。
- 至少 1 个“物件细节”（票根、封条、钥匙串、指纹、血渍、工牌）。
- 结尾必须留钩子（门缝、脚步、短信、章、远处警笛、风向变了）。
- 允许 20–30% 的情绪关键点做“化用”（对齐：`literaryAdaptation`），但必须克制、贴世界、不要硬贴名句。

### 3.4 `worldSetting`：世界是会自己走的

必须包含：

- **空间纹理**：世界由哪些“可重复细节”构成（让玩家能靠观察做决定）
- **势力格局**：至少 3 方（或 2 方 + 1 条制度链）并给出各自“想要什么”
- **规则与代价**：力量/信息/资源如何流动；违反会发生什么（可验证）
- **独立推进**：给出 2–3 个固定时钟或进程（季节、会议、抽查、粮尽、潮汐、伤口感染）

要点：

- 不写“主角是关键节点而非宇宙中心”这类 meta 句；直接写出“别人会怎么做”。
- 世界推进要像物理：不用威胁口吻，用机制陈述。

## 4) 反模板与禁用项（写作卫生）

### 4.1 禁止设计者口吻

禁止出现：

- “下一周目/复刷/重玩/通关/存档/任务面板/系统提示”
- “玩家必须…才能…”（改成制度与后果）
- 大段“主线一般沿…路线冲突…”的策划文体（改成世界事实与行动线）

### 4.2 禁止跨题材万能句复读

同一套句式不要在数百条主题里反复出现；每个主题要有自己的词汇域与镜头物件。

### 4.3 `humanizer-zh` 写作卫生（去 AI 感）

- 删掉“填充词”和“拐杖连接词”（如“此外/同时/因此/总之/值得注意的是”），能不写就不写。
- 少用可被摘出来当签名的“金句”。需要立场时，用事实与细节压出来，不用口号托底。
- 少用“不是X而是Y”“不仅…还…”这种对称句型；读起来太像模板。
- 列举尽量避免“强凑三项”。两项或四项更自然。
- 破折号/长横线少用。能用逗号句号说清，就别靠“一刀切断”的语气。

## 5) IP 主题（《》）的特别规则

- **不杜撰**：不确定的设定不要补成“看起来对”；宁可抽象到“气质与机制”，也不要编造具体事实。
- **保留原作气味**：核心冲突、叙事口吻、关键物件应贴近原作（但仍遵守“不读心、不代替玩家”）。
- 若需要补全世界细节，优先从“制度/地理/职业流程/物件纹理”补，少补“硬设定名词”。

---

## 6) 逐条修改执行方式（建议）

1. 先挑 1–3 条最差样例（占位符/复读句/系统腔）做“满意版本”对齐风格。
2. 以 20 条为一个批次逐条润色（可复用 `scripts/split_themes.cjs` 做分批对照）。
3. 每批次完成后做一次“禁用项扫描”（系统词、【A/B】占位符、复读句）。
4. 全部完成后再做一次一致性检查：JSON 可解析、字段齐全、换行与引号转义正确。
